- Expression becomes statement w/ an ;.
	- Expression statements: expression evaluted result discarded.
- Null statements: just a semicolon.
	- Useful when language requires statement, program logic doesn't need it.
	- Be careful b/c extra null statements aren't always harmless.
- Compound statement: 
	- Known as a block, sequence of statements in a pair of braces.
	- Names inside block: visible in that block and nested blocks.
	- Used when language requires single statement, logic has more than one.
		- While loop: body is one, but we need to execute multiple.
- Variables can be declared within control structure of conditional/while statements.
	- Variables defined: only within statement, out of scope after statement ends.
		- So within parenthesis of while loop, etc. and the body.
		- If access needed outside, declare outside the loop.
- Conditional statements:
	- `if`: determine the flow of control based on a condition.
	- `switch`: evaluate integral expression and choose an execution path.
	- if:
		- condition can be expression or initialized variable declaration. 
		- Expression/variable needs to be convertible to bool.
	- Just always use curly braces.
		- Without curly braces keep in mind that for a conditional/else, the next line is tested but the rest after will always be executed. 
	- How to solve dangling `else` problem:
		- Match each else with the closest preceding if.
	- switch:
		- evaluate parenthesized expression following switch keyword.
		- expression:  initialized variable declaration, converted to integral type.
		- break transfers control out of a switch.
			- Returns control to the enclosing while.
		- no match found:
			- execution falls through to the first statement following the switch.
		- case labels: integral constant expressions.
		- can't have two case labels w/ the same value.
		- after case label matches, execution starts at that label, continues across all remaining cases until program explicitly interrupts it.
			- if no break, then it falls through multiple case label.
		- default label: executed when no case label matches switch expression value.
		- what if skipped code has a variable definition?
			- illegal to jump from place where variable w/ initializer is out of scope to where variable is in scope.
			- this is because all case labels share the same scope.
				- If u skip an initializer causes dangerous situation here operations are done on objects assumed to be initialized but not actually. 
			- put declarations in braces.
	- iterative statements:
		- loops: enable repeated execution until condition is true.
		- while and for: test condition before body
		- do while executes body and then tests condition.
		- while loop:
			- variables defined within while loop are created and destroyed on every iteration.
			- use when don't know the total number of iterations something will run.
		- for loop:
			- `for (init-statement, condition, expression`):
				- statement.
			- expression: evaluated after each iteration of the loop.
			- Process:
				1. Init-statement: executed once at the start of the loop
				2. Condition evaluated (if false, loop body not executed at all)
				3. Body executes.
				4. Expression evaluated.
			- All subsequent runs do 2, 3, 4.
			- all objects in the for header are visible/limited to only body of the loop.
			- can pass in null statement for any part.
				- have a init outside
				- omitting condition: same as writing true for condition.
					- for body must exit loop
			- range for loop:
				- `for (declaration : expression): statement`
					- expression is a sequence or an object w/ iterators.
					- control variable must be of type as expression or convertible.
					- execution ends when all elements are processed.
					- can't add elements to vector while iterating over w/ range (end iterator is cached so gets invalidated)
			- do-while: condition is tested after statement body completes.
				- ```c++
				  do {statement} while (condition)
				  ``` 
				- statement is evaluated before condition is evaluated. 
				- thing inside while can't be empty.
				- can't define variables inside the condition.
					- this would allow for variable use before definition.  
		- jump statements: 
			- interrupts flow of execution.
			- break statement: terminates nearest `while, do while, for, switch`
				- only nearest, need multiple breaks for a switch.
			- continue statement: terminates current iteration of the nearest enclosing loop, begins next iteration.
				- only in a `for, while, do while` loop.
				- can only be in a switch if the switch is in an iterative statement.
					- while, do while: evaluate the condition
					- traditional for, continues at `expression`.
					- range for: continues by initializing control variable from next element. 
			- goto statement:
				- `goto label`.
					- label is an identifier for a statement.
					- labels can have same identifier as another entity (variable)
					- cannot transfer control from a point where variable in scope vs out of scope.
					- jump backward is okay: destroys variable and reconstructs it.
		- Exceptions:
			- run-time anomalies
				- throw raises an exception (sends it up), but doesn't actually resolve it.
			- try blocks:
				- enclose dealing with exception.
					- ends with one or more catch clauses.
					- actually handles the error.
			- exception classes: pass info about throw and catch.
			- try:
				- ```c++
				  try {
					  //program
				  } catch (exception-declaration) {
					  handler
				  }
				  ```
				- after catch handles an exception, exceution goes to statement following last catch clause. 
					- all exception classes have a member function `what`:
					- returns a C-style character string used to initialize the exception in the `throw`.
				- exception thrown: function that threw it is searched first.
				- no matching catch: function terminates, go up the stack.
					- if no program found, go to a library function called `terminate`.
				- no try blocks, go up in call stack, then call `terminate`.
			- standard exception classes:
				- located in `stdexcept` header, throw objects of these. 
				- can only default initialize `exception`, `bad_alloc`, `bad_cast`.
					- not possible to provide initializer.
				- for all else, initialize w/ a `string`or `C-style` string but can't default initialize them. 
					- string provides specific information about what error happened.
					- objects can access `what`, returns a `const char *` to provide textual description of thrown error. 
						- This is what we initalized with, but for other types, varies by compiler.
